# План исправления проблем в проекте

## Обзор проекта
Проект представляет собой веб-приложение на FastAPI для обработки Excel файлов, классификации номеров и сравнения данных. Основные компоненты:
- `main.py` - FastAPI сервер с API endpoints
- `comparator.py` - бизнес-логика обработки файлов
- `database.py` - работа с SQLite базой данных
- `templates/index.html` - фронтенд интерфейс

## Выявленные проблемы

### 1. Проблема с предобработкой файла 2
**Симптом**: Фронтенд не показывает варианты в таблице сопоставления после изменений в `preprocess_second_file` и `get_unique_pabricushki`.

**Причины**:
- Ошибка `'final_col_idx is not defined'` (возможно уже исправлена, не найдена в текущем коде)
- Несоответствие структуры файла после предобработки
- Возможно пустой `pab2_list` при передаче во фронтенд

### 2. Дублирующиеся обработчики в HTML
**Проблема**: В файле `templates/index.html` есть два обработчика для кнопки `confirmMultiSelectBtn` (строки 1145 и 1554), что вызывает двойное срабатывание.

**Решение**: Удалить первый обработчик (строки 1145-1170), так как второй обработчик более полный и обрабатывает оба случая (пабрикушки и поля).

### 3. Отладочные print() в коде
**Проблема**: В `comparator.py` есть несколько отладочных print() вызовов, которые нужно убрать перед продакшеном.

**Расположение**:
- Строка 347: предупреждение о чтении справочника для типа К
- Строки 749-752: сообщения о завершении обработки
- Строки 772-782, 792: отладочная информация в `get_unique_pabricushki`
- Строка 876: предупреждение в `preprocess_second_file`
- Строки 939-945: отладочная информация в `preprocess_second_file`
