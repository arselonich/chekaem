# Деплой Excel Comparator на публичный сервер

## Описание
Это руководство описывает процесс деплоя приложения для объединения и сравнения Excel файлов на публичный сервер.

## Изменения для публичного доступа
Приложение было модифицировано для работы в публичной среде:
- Сервер запускается на `0.0.0.0` вместо `127.0.0.1`
- Файлы сохраняются в папку `uploads/` проекта вместо `~/Downloads`
- Добавлен CORS middleware для кросс-доменных запросов
- Добавлена проверка размера загружаемых файлов (макс. 100 MB)
- Реализована очистка старых сессий (24 часа)

## Варианты деплоя

### 1. Docker Compose (рекомендуется)

#### Требования
- Docker
- Docker Compose

#### Шаги
1. Скопируйте все файлы проекта на сервер
2. Настройте доменное имя (опционально)
3. Запустите приложение:
   ```bash
   docker-compose up -d
   ```

4. Приложение будет доступно на порту 80 (HTTP)

#### Настройка HTTPS
1. Создайте папку `ssl/` и поместите туда SSL сертификаты
2. Раскомментируйте HTTPS секцию в `nginx.conf`
3. Перезапустите nginx:
   ```bash
   docker-compose restart nginx
   ```

### 2. Systemd Service (без Docker)

#### Требования
- Python 3.9+
- Установленные зависимости: `pip install -r requirements.txt`

#### Шаги
1. Скопируйте проект в `/opt/excel-comparator`
2. Создайте виртуальное окружение:
   ```bash
   cd /opt/excel-comparator
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. Создайте необходимые директории:
   ```bash
   mkdir -p uploads
   chown -R www-data:www-data uploads
   ```

4. Скопируйте service файл:
   ```bash
   sudo cp excel-comparator.service /etc/systemd/system/
   ```

5. Запустите сервис:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable excel-comparator
   sudo systemctl start excel-comparator
   ```

6. Настройте nginx как reverse proxy (пример конфигурации в `nginx.conf`)

### 3. Ручной запуск

```bash
# Установите зависимости
pip install -r requirements.txt

# Инициализируйте базу данных
python -c "from database import init_db; init_db()"

# Запустите сервер
HOST=0.0.0.0 PORT=8000 python main.py
```

## Конфигурация

### Переменные окружения
- `HOST` - хост для запуска (по умолчанию: `0.0.0.0`)
- `PORT` - порт (по умолчанию: `8000`)
- `MAX_FILE_SIZE` - максимальный размер файла в байтах (в коде)

### Безопасность
1. **Ограничение доступа**: Настройте firewall для доступа только к необходимым портам
2. **HTTPS**: Настоятельно рекомендуется использовать HTTPS в продакшене
3. **Аутентификация**: Для публичного доступа рассмотрите добавление базовой аутентификации
4. **Мониторинг**: Настройте логирование и мониторинг

## Мониторинг и обслуживание

### Логи
- Логи приложения: `server.log`
- Логи nginx: `/var/log/nginx/access.log` и `/var/log/nginx/error.log`
- Systemd логи: `sudo journalctl -u excel-comparator -f`

### Очистка старых файлов
Приложение автоматически очищает сессии старше 24 часов. Для ручной очистки:
```bash
# Удаление файлов старше 7 дней
find uploads/ -type f -mtime +7 -delete
find uploads/ -type d -empty -delete
```

### Резервное копирование
Рекомендуется регулярно создавать резервные копии:
- База данных: `data.db`
- Конфигурационные файлы

## Устранение неполадок

### Сервер не запускается
1. Проверьте логи: `docker-compose logs` или `journalctl -u excel-comparator`
2. Убедитесь, что порт не занят: `netstat -tulpn | grep :8000`
3. Проверьте права доступа к папке `uploads/`

### Загрузка файлов не работает
1. Проверьте размер файла (макс. 100 MB)
2. Убедитесь, что папка `uploads/` существует и доступна для записи
3. Проверьте настройки nginx `client_max_body_size`

### Проблемы с базой данных
1. Убедитесь, что файл `data.db` существует
2. Проверьте права доступа: `chmod 664 data.db`
3. Переинициализируйте БД: `python -c "from database import init_db; init_db()"`
