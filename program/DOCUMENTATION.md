## Общее описание

Приложение предназначено для автоматизированной классификации, сравнения и объединения данных о пабрикушках (производственных объектах) и названиях полей из двух исходных Excel-файлов с использованием третьего файла-справочника. Результатом является итоговый Excel-отчёт с детальным анализом соответствий и расхождений.

### Ключевые возможности

- **Классификация номеров 3 типов:**
  1. **Тип Э** — номера формата XX-XX-XX (начинаются с цифры, извлекается до второго дефиса)
  2. **Тип К** — номера с тремя двоеточиями между цифрами (например, `123:456:789:012`)
  3. **Тип Склад** — номера, содержащие слово "Склад" (исключаются из обработки)
  4. **Неопределённые** — требуют ручной классификации пользователем

- **Предобработка файла 2** при анализе — заполнение пустых названий полей из справочника, добавление финальных номеров

- **Двухэтапное сопоставление:**
  1. Сопоставление пабрикушек (производственных объектов)
  2. Сопоставление названий полей

- **Авто-сопоставление** на основе fuzzy matching (нечёткого поиска)
- **Сохранение соотношений** в базе данных для повторного использования
- **Исключение нежелательных записей** (семена, НЕ ВХОДИТ В ПАТ, не учитываем!!)
- **Генерация отчёта** в формате Excel с детальным сравнением
- **Модификация файлов сессии** — после обработки файлы в папке сессии обновляются с заполненными данными

## База данных

### Схема БД (SQLite)

#### Таблица `places`
Хранит список мест (локаций/хозяйств)

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | INTEGER | Первичный ключ (AUTOINCREMENT) |
| name | TEXT | Уникальное название места (1, 2, 3, 4, 5, ...) |

#### Таблица `mappings`
Соотношения пабрикушек для каждого места

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | INTEGER | Первичный ключ |
| place_name | TEXT | Название места (внешний ключ) |
| pab1 | TEXT | Название пабрикушки из Файла 1 |
| pab2 | TEXT | Соответствующее название из Файла 2 |
| is_excluded | INTEGER | Флаг исключения (1 = исключено) |

**Уникальность:** `UNIQUE(place_name, pab1)`

#### Таблица `field_mappings`
Соотношения названий полей для каждого места

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | INTEGER | Первичный ключ |
| place_name | TEXT | Название места (внешний ключ) |
| field1 | TEXT | Название поля из Файла 1 |
| field2 | TEXT | Соответствующее название из Файла 2 |
| is_excluded | INTEGER | Флаг исключения (1 = исключено) |

**Уникальность:** `UNIQUE(place_name, field1)`

#### Таблица `number_classifications`
Пользовательские классификации номеров для каждого места

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | INTEGER | Первичный ключ |
| place_name | TEXT | Название места (внешний ключ) |
| raw_number | TEXT | Исходный номер из файла |
| number_type | TEXT | Тип номера: 'Э', 'К', или 'Склад' |
| extracted_value | TEXT | Извлечённое значение для поиска в справочнике |

**Уникальность:** `UNIQUE(place_name, raw_number)`

## Вычислительные процессы

### 0. Предобработка файла 2 (НОВАЯ)

**Функция:** `preprocess_second_file(file_path, sprav_file)`

**Когда вызывается:** При анализе файлов (`POST /api/analyze`), ДО показа таблиц сопоставления.

**Что делает:**
1. Читает ВЕСЬ файл 2 целиком (сохраняя структуру: заголовки + данные)
2. Читает справочник (файл 3)
3. Для каждой строки данных:
   - Классифицирует исходный номер (столбец F) → тип Э/К
   - Находит финальный номер в справочнике
   - **Добавляет новый столбец справа** с финальным номером
   - Если название_номера (столбец D) пустое → заполняет из столбца F справочника
4. Сохраняет модифицированный файл обратно с именем листа "Лист1"

**Структура файла 2 после предобработки:**
```
Столбцы 0-5 (исходные): A(пабрикушка), B(вес), C, D(название_номера), E, F(исходный_номер)
Столбец 6 (новый): финальный_номер
```

**Зачем:** Последующие этапы (сопоставление полей, обработка) работают уже с заполненными данными. Пользователь видит корректные уникальные названия полей.

### 1. Классификация номеров

**Функция:** `classify_number(raw_num: str) -> Tuple[NumberType, Optional[str]]`

**Типы номеров:**

| Тип | Критерий | Извлечение | Пример |
|-----|----------|------------|--------|
| **Э** | Начинается с цифры + 2+ дефиса между цифрами | До второго дефиса | `"12-34-56"` → `"12-34"` |
| **К** | Содержит число с 3 двоеточиями | Число между двоеточиями | `"(OP-323) 22667:2303323:92304923049234:4324234"` → `"22667:2303323:92304923049234:4324234"` |
| **Склад** | Содержит слово "Склад" (регистронезависимо) | Не извлекается | `"Склад №5"` → `None` |
| **Неопределённый** | Не подошёл ни под один тип | Не извлекается | `"ABC123"` → `None` |

### 3. Чтение справочника

**Функция:** `read_spravochnik(file_path)`

**Параметры:**
- Лист: `"кадастровые номера полей"`
- Столбцы:
  - U (индекс 20) → `список_номеров` (номера через запятую)
  - D (индекс 3) → `финальный_номер`
  - F (индекс 5) → `название_номера_справочник` (НОВОЕ)

**Создание словаря соответствия:**
```python
lookup_e = {}  # Для типа Э
field_name_lookup = {}  # {финальный_номер: название_номера}
for _, row in sprav_df.iterrows():
    numbers = row['список_номеров'].split(',')
    final_num = row['финальный_номер']
    field_name = row['название_номера_справочник']
    for num in numbers:
        num = num.strip()
        if num:
            lookup_e[num] = final_num
    if final_num and field_name:
        field_name_lookup[final_num] = field_name
```

**Для типа К** (столбец K → D + F):
```python
lookup_k = {}  # Для типа К
# Читается из отдельного файла справочника
for _, row in sprav_k.iterrows():
    k_val = row.iloc[10]  # Столбец K
    d_val = row.iloc[3]   # Столбец D
    f_val = row.iloc[5]   # Столбец F (название)
    if pd.notna(k_val) and pd.notna(d_val):
        lookup_k[str(k_val).strip()] = str(d_val).strip()
        if pd.notna(f_val):
            field_name_lookup[str(d_val).strip()] = str(f_val).strip()
```

### 4. Чтение второго файла (Лист1) с классификацией

**Функция:** `read_second_file(file_path, sprav_df, number_classifications)`

**Важное изменение:** Файл 2 уже модифицирован функцией `preprocess_second_file`. Номера уже классифицированы, названия заполнены, финальные номера добавлены.

**Последовательность обработки:**

```
1. Классификация каждого номера
   ├─ Проверка пользовательских классификаций (из БД)
   └─ Автоматическая классификация через classify_number()

2. Исключение типа "Склад" → df_sklad

3. Исключение типа "Неопределённый" → df_undefined

4. Поиск финального номера в справочнике
   ├─ Тип Э → lookup_e (столбец U)
   └─ Тип К → lookup_k (столбец K)

5. Заполнение пустых названий из справочника (НОВОЕ)
   └─ Если название_номера пустое → берётся из field_name_lookup по финальному номеру

6. Разделение на df_ok (найден) и df_errors (не найден)
```

### 5. Получение неопределённых номеров

**Функция:** `get_undefined_numbers(second_file, saved_classifications=None)`

**Изменение:** Теперь принимает `saved_classifications` — словарь сохранённых классификаций из БД. Статистика считается по ВСЕМ номерам с учётом сохранённых классификаций.

**Алгоритм:**
```python
stats = {'Э': 0, 'К': 0, 'Склад': 0, 'Неопределённый': 0, 'Всего': len(df)}
undefined_set = set()

for raw_num in df['исходный_номер']:
    # Сначала проверяем сохранённые классификации
    if saved_classifications and raw_num in saved_classifications:
        num_type = saved_classifications[raw_num][0]
        stats[num_type] += 1
    else:
        # Автоматическая классификация
        num_type, _ = classify_number(raw_num)
        stats[num_type] += 1
        if num_type == 'Неопределённый':
            undefined_set.add(raw_num)

return sorted(list(undefined_set)), stats
```

### 6. Fuzzy Matching (нечёткий поиск)

**Функция:** `fuzzy_match(pab1, pab2_list, threshold=0.6)`

**Алгоритм:**
```python
from difflib import SequenceMatcher

def fuzzy_match(pab1, pab2_list, threshold=0.6):
    best_match = None
    best_score = threshold

    for pab2 in pab2_list:
        if not pab2:
            continue
        score = SequenceMatcher(None, pab1.lower(), pab2.lower()).ratio()
        if score > best_score:
            best_score = score
            best_match = pab2

    return best_match
```

**Результат:** Одинаковые ключи → данные объединяются

### 8. Агрегация данных

**Функция:** `aggregate_by_key(df, source_name)`

**Группировка:**
```python
grouped = df.groupby(['ключ', 'пабрикушка'])['вес'].sum().reset_index()

pab_dict = {}  # {ключ: [список пабрикушек]}
weight_dict = {}  # {ключ: {пабрикушка: суммарный_вес}}

for key in grouped['ключ'].unique():
    subset = grouped[grouped['ключ'] == key]
    pab_dict[key] = sorted(subset['пабрикушка'].tolist())
    weight_dict[key] = dict(zip(subset['пабрикушка'], subset['вес']))
```

### 9. Построение итоговой таблицы

**Функция:** `build_result_table((pab1, w1), (pab2, w2), g_by_key)`

**Колонки итогового DataFrame:**

| Колонка | Название | Содержание |
|---------|----------|------------|
| A | Номер | левая часть ключа (до `|`) |
| B | Значения из столбца I | переработанные номера (уникальные через `; `) |
| C | Название номера | правая часть ключа (после `|`) |
| D | Пабрикушки из Данные1.1 | список пабрикушек из 1-го файла (через `; `) |
| E | Вес из Данные1.1 | `Пабрикушка - вес` (с переносами `\n`) |
| F | Пабрикушки из Данные2 | список пабрикушек из 2-го файла (через `; `) |
| G | Вес из Данные2 | `Пабрикушка - вес` (с переносами `\n`) |
| H | Статус | один из 4 статусов (см. ниже) |
| I | Есть в 1, нет во 2 | пабрикушки только в 1-м файле |
| J | Есть во 2, нет в 1 | пабрикушки только во 2-м файле |
| K | Расхождения по весам | `Пабрикушка: вес1 vs вес2 (разница)` |

**Статусы (колонка H):**

| Статус | Условие |
|--------|---------|
| `"все хорошо"` | составы совпадают, веса совпадают |
| `"состав одинаковый, но есть расхождения веса"` | составы совпадают, веса различаются |
| `"состав разный"` | составы различаются, веса совпадают |
| `"состав разный и есть расхождения веса по общим позициям"` | составы различаются, веса различаются |

### Статистика классификации номеров

**Обновлённая логика:**

1. При загрузке неопределённых номеров бэкенд возвращает **полную** статистику по ВСЕМ номерам (с учётом сохранённых классификаций из БД)

3. После сохранения пользовательских классификаций статистика **перезагружается с сервера** (а не пересчитывается вручную)
4. Прогресс-бары показывают процент от общего количества ВСЕХ строк

## История изменений

### Версия 2.0 (текущая)

#### Новые функции

1. **Предобработка файла 2 при анализе**
   - Функция `preprocess_second_file()` вызывается при `POST /api/analyze`
   - Заполняет пустые названия полей из справочника (столбец F)
   - Добавляет столбец с финальными номерами справа
   - Сохраняет модифицированный файл обратно в сессию

2. **Обновлённая статистика классификации номеров**
   - `get_undefined_numbers()` принимает `saved_classifications` из БД
   - Статистика считается по ВСЕМ номерам (автоматические + сохранённые)
   - Фронтенд перезагружает статистику с сервера после сохранения
   - Удалена переменная `autoStats`, используется единая `numberStats`

3. **Заполнение пустых названий в `read_second_file`**
   - Если `название_номера` пустое → берётся из `field_name_lookup` по финальному номеру
   - Работает для обоих типов номеров (Э и К)

4. **Модификация файлов сессии после обработки**
   - `process_files` сохраняет модифицированные file1.xlsx и file2.xlsx
   - Файлы содержат заполненные названия и финальные номера

#### Исправления

1. **Совпадение ключей между файлами**
   - Файл 1: номера уже финальные, не преобразуются
   - Файл 2: финальные номера из справочника
   - Ключи формируются одинаково: `номер|название_номера`

2. **Сохранение структуры файла 2**
   - `preprocess_second_file` читает весь файл целиком
   - Модифицирует существующие столбцы на месте
   - Новый столбец добавляется справа
   - Имя листа "Лист1" сохраняется

3. **Исправление `get_unique_fields`**
   - Читает уже модифицированный файл 2
   - Возвращает заполненные названия полей

Данное приложение представляет собой полнофункциональную систему для классификации, сравнения и объединения данных из Excel-файлов с использованием:

- **FastAPI** для бэкенда
- **SQLite** для хранения соотношений и классификаций
- **pandas** для обработки данных
- **Fuzzy matching** для авто-сопоставления
- **Регулярные выражения** для классификации номеров
- **Современный JavaScript** для интерактивного интерфейса

### Ключевые улучшения версии 2.0

1. **Предобработка на этапе анализа** — файл 2 модифицируется сразу при загрузке, что позволяет пользователю видеть уже заполненные данные
2. **Корректное совпадение ключей** — оба файла используют финальные номера из справочника
3. **Полная статистика классификации** — учитывает как автоматические, так и сохранённые в БД классификации
4. **Модификация файлов сессии** — после обработки файлы содержат заполненные данные для проверки
